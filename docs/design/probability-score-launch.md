## Vision

### 10x Check
The 10x version is a FIRE co-pilot connected to real account data (Plaid integration),
with a probability score that updates daily as actual spending changes. "Your FIRE
probability dropped 2% this month — you overspent by $400 vs plan." Not a calculator:
a live financial nervous system that nudges behavior every day.

### Approach Selected
C (Parallel tracks): Build probability score + fix polish bugs + post Reddit this week.
The Reddit post can be drafted on Day 1 independent of all engineering tasks.
Feature work proceeds on Days 1-5 in the sequence below.

  - Update all 6 `calcFIRE()` call sites in `app/page.tsx` (lines 883, 934, 937-940) to pass `initialBalance` (default 0)

**Analytics (Day 2):**
- Delete `instrumentation-client.ts` (bare init pattern is fragile)
- Add `<PostHogProvider apiKey={process.env.NEXT_PUBLIC_POSTHOG_KEY}>...</PostHogProvider>` wrapping `{children}` in `app/layout.tsx`
- Add `posthog.capture('wizard_step_completed', { step })` at each `onNext` in `app/page.tsx` (screens: hero, city, income, savings)
- Fire `posthog.capture('calculator_completed', { city, fireTarget, years })` in RevealScreen when `revealed` transitions to `true`
- Verify `NEXT_PUBLIC_POSTHOG_KEY` is set in Vercel project settings before deploy

**Monte Carlo probability score (Days 2-3):**
- Create `lib/monte-carlo.ts` exporting `monteCarloFIRE(inputs, runs=500)`:
  - Guard: if any required input is NaN/undefined/null → return `{ p10: null, p50: null, p90: null, probability: null }`
  - Per run: sample growthRate from uniform distribution over [0.01, 0.13] (mean 7% ± 6%)
  - Count runs where portfolio reaches `fireTarget` within 40 years
  - `probability` = count / runs (fraction of runs that reach FIRE within 40 years)
  - Return `{ p10: year, p50: year, p90: year, probability: number }`
  - 500 runs (not 1,000) to stay comfortably on main thread (~1.25ms on mobile)
  - Add TODO comment for Web Worker migration if profiling shows jank
- Hoist `calcProjection()` to parent `Page` component (`app/dashboard/page.tsx`):
  - Move both useMemo calls (lines 172 and 589) to the top-level Page component
  - Pass `{ data, fireYear, fireTarget, annualSavings }` as props to DashTab and FIRETab
  - `fireAge` already lives in parent (line 845); `retireAge = fireAge + (fireYear ?? 0)` computed in parent
- Call `monteCarloFIRE()` from parent Page component (separate useMemo, debounced 300ms for slider)
- Pass `{ p10, p50, p90, probability }` to DashTab as props
- Display on Overview tab, **above** the existing KPI grid:
  - Large probability number ("73%")
  - Subtitle "likely to retire by [p50 year]"
  - Small range: "best case: [p10] / base: [p50] / worst case: [p90]"
  - If probability = null: show "— Add your income to see your FIRE probability"

**Location/tax selector for custom cities (Day 4):**

  - Show a searchable dropdown of ALL STATE_TAX entries from `lib/fire-data.ts`
  - Entries include US states and international countries (already exist in STATE_TAX)
  - STATE_TAX labels are already human-readable ("California — ~7.20% effective")
  - On selection: set `stateKey` from selected entry
  - Default selection: no default — user must pick. Show placeholder "Select your state or country"
  - `STATE_TAX[stateKey] ?? { rate: 0, label: 'Unknown' }` already has null safety (line 485)

**What-if savings slider (Day 4-5):**
- Location: dashboard Overview tab, below the probability score display
- Label: "If you saved $X/month instead →"
- Range: $0–min(income, $10,000)/month; step $100; default = `Math.round(annualSavings / 12 / 100) * 100`
- Slider cap: max = income (can't save more than you earn) — compute from hoisted `annualSavings` / monthly income
- On change: debounce 300ms → rerun `monteCarloFIRE()` with adjusted savings rate ONLY
  - Base `calcProjection()` and chart are NOT re-run (D3: slider updates probability score only)
- Delta display: "→ 78% likely (+5%)" in teal if higher, "→ 65% likely (-8%)" in red if lower
- Fire `posthog.capture('fire_slider_adjusted', { newMonthlySavings, probabilityDelta })` on debounce
